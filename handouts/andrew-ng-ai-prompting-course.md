## 課程概述

這門課由 Andrew Ng（DeepLearning.AI 創辦人）主講，目標是讓學習者從 AI 新手進化到 **AI Power User**。課程分三個模組：

| 模組 | 主題 | 核心問題 |
|------|------|----------|
| Module 1 | AI 基礎與資訊搜尋 | AI 怎麼知道它知道的事？如何讓它找到對的資訊？ |
| Module 2 | AI 作為思考夥伴 | 如何讓 AI 幫你思考、寫作、批評？ |
| Module 3 | 超越文字的 AI | 如何用 AI 生圖、分析資料、建 App？ |

**本課程寫作時間：2026 年**，部分建議已比 2022-2023 年的傳統 prompting 建議更新（例如：不再需要「think step by step」）。

### 1. AI 新手 vs AI 進階用戶

#### 核心論點

Andrew Ng 用五個維度對比新手和進階用戶的差異：

**① 問題的難度**
- **新手**：把 AI 當搜尋引擎，問簡單問題（"Does Taco Bell still have the double deck taco?"）
- **進階用戶**：丟進大量文件（車子規格、保險方案、報價），讓 AI 花幾分鐘深度分析後給出完整報告

**② 提供 Context 的多寡**
- **新手**：給一個短 prompt，期望 AI 自己填空
- **進階用戶**：把 AI 當成「剛入職的聰明大學畢業生」——他很聰明，但什麼都不知道，你要給他足夠背景才能做好事情
  - 例：不只說「幫我寫自我考核」，而是上傳專案追蹤截圖、最近的文件、語音備忘錄，再叫 AI 根據這些寫

**③ 如何獲得誠實的回饋**
- **新手**：問「我有個很棒的商業點子，幫我批評一下」→ AI 會說「真棒！」
- **進階用戶**：
  1. 用中立的語氣提問，不透露自己期望的答案
  2. 或提供 **rubric（評分標準）**：「用以下標準評估：有沒有問題存在？有沒有市場？我有競爭優勢嗎？」
  - 結果：AI 的答案從「太棒了」變成「這個想法 8/100，原因如下…」

**④ 寫作工作流程**
- **新手**：「幫我寫一篇關於黑莓機的部落格」→ 得到 AI 味重的平庸文章
- **進階用戶**：
  1. 先要求 AI 列出大綱
  2. 反覆來回批評、修改大綱
  3. 大綱確定後再展開成重點段落
  4. 段落確定後才寫最終文章
  - AI 此時是「思考夥伴」，不是「一次性生成機器」

**⑤ 對 AI 犯錯的認知**
- 新手因為看到「草莓有幾個 R」這類病毒式失敗案例，高估了 AI 犯錯的頻率
- 進階用戶知道現在的 AI 已能做：深度研究報告、個人健康數據分析、建立網站

#### 關鍵心智模型

> **把 AI 想成一個剛入職的聰明大學生：高度積極、知識廣博，但對你一無所知。你提供的 context 越好，他做的越好。**

### 2. AI 的知識來源：Pre-trained Knowledge

#### 核心論點

AI 從「讀」大量網路文字中學習，這些學來的知識叫 **pre-trained knowledge**（預訓練知識）。

**AI 讀過什麼？**
- 社群媒體（Reddit、Quora）
- 書籍、百科（Wikipedia）
- 新聞網站
- 學術論文
- 以上合計「數兆個字」

**知識的「密度」決定 AI 的可靠程度**

| 主題類型 | 網路資料量 | AI 的可靠度 |
|----------|-----------|-------------|
| 料理、電影、名人 | 極多 | 高 |
| 常見事實（最高建築） | 多 | 高 |
| Quasar（天文專有名詞） | 少 | 中等 |
| 你公司的內部文件 | 零 | 無法回答 |

**語言的比例**
- 大多數是英文，其他語言比例極低
- 廣東話使用者超過 8,000 萬人，但廣東話網路內容可能不到 0.1%

**AI 對打字錯誤的容忍度**
- 能理解 "cn u cook eggs in mcrowave"（大量錯字）
- 原因：訓練資料本身就有大量錯字
- **實際建議**：打字時不必太在意小錯字，直接送出即可

**Pre-trained knowledge 的限制**
- 含有誤解和過時資訊
- 訓練在某個日期截止，之後的事情不知道 → 這就是為什麼需要 web search

### 3. Web Search：即時資訊的獲取

#### 核心論點

AI 的 **knowledge cutoff date**（知識截止日期）之後發生的事，只能靠 web search 補充。

**什麼時候 AI 會觸發 Web Search？**

| 情境 | 是否用 Web Search |
|------|-----------------|
| "I dropped my phone in soup. What to do?" | 否（常識即可） |
| "What's the 67 meme from 2025?" | 是（很可能在截止日之後） |
| "Find me a highly rated gym near Mountain View" | 是（即時性 + 地點特定） |
| "What's on Voyager 1's vinyl record?" | 否（不依賴即時資訊） |

**Web Search 的兩種觸發方式**
1. AI 自己判斷要不要搜
2. 用戶在 prompt 中明確要求：「請先做 web search 再回答」

**Web Search 的限制（重要！）**
1. **容易引用社群媒體**：根據一份報告，AI 最常引用的來源第一名是 Reddit，依序是 Wikipedia、YouTube 等 → 不一定可靠
2. **可能引用過時網頁**：曾有人用 AI 找跑步地點，結果它引用了 20 年前的網頁，那個學校現在已不對外開放
3. **AI 沒有完整讀過它引用的文章**：背後有兩個 AI 模型在合作，第一個 AI（你在對話的那個）只收到「摘要」，沒有讀完整文章 → 有時會誤解原文

**如何讓 AI 使用更可靠的來源**

直接在 prompt 中指定：
```
請只使用官方健康機構（WHO、FDA、歐洲藥品管理局）的資訊來回答這個問題
```

**Web Search vs 直接用搜尋引擎**

| 需求 | 適合工具 |
|------|---------|
| 快速掃描多個來源、找特定網站 | 搜尋引擎（Google / Bing） |
| 想要多個來源的「綜合結論」、有取捨的判斷 | AI + Web Search |

### 4. Deep Research：深度多來源研究

#### 核心論點

**Deep Research** 是 AI 模型的一種特殊模式，能自主搜尋數十甚至上百個來源，花數分鐘到數小時進行深度分析後給出報告。ChatGPT、Gemini、Claude 都有這個功能。

**Deep Research vs 基本 Web Search 的差異**

| 維度 | 基本 Web Search | Deep Research |
|------|---------------|---------------|
| 來源數量 | 少數幾個 | 數十個以上 |
| 資訊即時性 | 可取得最新資訊 | 可取得最新資訊 |
| 回答時間 | 數秒到數十秒 | 數分鐘以上 |
| 適用情境 | 單一問題，幾秒內能回答的事 | 複雜問題，需要整合多元觀點 |

**三種資訊獲取方式的完整對比**

| 方式 | 適用情境 | 速度 |
|------|---------|------|
| Pre-trained knowledge | 常識問題、不需即時資訊 | 最快（秒） |
| Web Search | 即時資訊、地點特定查詢 | 中等（數秒到數十秒） |
| Deep Research | 複雜問題、需要多來源整合 | 最慢（數分鐘以上） |

**使用 Deep Research 的正確流程**
1. 給 AI 足夠的 context（例：我在 Palo Alto，前院 30x20 英呎，想辦萬聖節鬼屋）
2. AI 會先提出「研究計畫」（research plan）
3. 用戶審閱計畫，可以批評、修改
4. 確認後讓 AI 執行：自動搜尋 → 讀取 → 整合 → 輸出報告

**Andrew Ng 的評論**：Deep Research 是「嚴重被低估的工具」，很多人沒有善用它。

### 5. Context 管理：讓 AI 更懂你的任務

#### 核心論點

**Context（上下文）** 是 AI 用來生成回應的所有資訊，包含：system prompt、你的 prompt、對話歷史、上傳的文件。

**現代 AI 的 Context 容量**
- 人類工作記憶：約 7 件事
- 現代 AI：可接受高達約 750,000 字的 context（約前 4-5 本哈利波特）

**Context 的組成**（一個對話的預設 context 包含）
1. System prompt（AI 的基本指示：日期、名稱、能力、工具說明）
2. 你的 prompt
3. 對話歷史（你和 AI 的所有來回）
4. 上傳的文件

**Context 管理的核心技巧**

> **思考什麼資訊是「一個受信任的顧問需要知道的」才能回答好這個問題。**

**什麼時候要開新對話**

當你想問一個「跟目前對話完全無關的問題」時，開新對話可以清空 context，避免舊的 context 干擾新的問題。

例：你在討論自己的健身計畫，突然想問媽媽的健身計畫 → 開新對話，否則你自己的偏好和條件會影響 AI 為媽媽生成的建議。

#### AI Desktop Apps（代理式 Context 讀取）

- 工具代表：Claude Code/Cowork、Microsoft Copilot、Google Gemini
- 核心能力：AI 可以**自己去你的電腦裡找相關檔案**，不需要你手動上傳
- 例：告訴 AI「幫我整理這個資料夾」，AI 會自己瀏覽、讀取、重新命名、移動檔案

**使用 Desktop App 的安全注意事項**
1. 給 AI 存取**最小必要資料夾**，不要讓它讀取整個電腦
2. 仔細檢查它的「權限請求」
3. 被刪除的檔案**不會進回收桶**，無法還原
4. 被編輯的檔案通常**沒有編輯歷史**，無法回復
5. 建議流程：讓 AI 先提案 → 你審核 → 確認後再執行

### 6. Reasoning Models：讓 AI 深度思考

#### 核心論點

現代 AI 的一大突破是 **reasoning（推理）** 能力——它可以花很長的時間思考，完成以前需要人類數小時的複雜任務。

**AI 能完成任務的複雜度演進**

| 時期 | AI 能完成的任務難度 |
|------|-----------------|
| 早期（2022-2023）| 幾秒鐘能完成的任務 |
| 2024-2025 | 幾秒到幾分鐘的任務 |
| 現在（2026） | 人類需要數小時的任務 |

**Reasoning 的運作邏輯**

```
你的 prompt
    ↓
AI 開始思考
    ↓
[可能] 需要更多資訊 → 使用工具（web search / 讀取檔案）
    ↓
繼續思考
    ↓
[重複幾次]
    ↓
輸出最終答案
```

**如何觸發 Reasoning**
- 在介面上選擇「thinking mode」/ 「thinking option」
- 或在 prompt 中直接說：「think really hard about this」/ 「ultra think」

**已過時的建議**
- ❌ 「Tell the AI to think step by step」→ 在 2022-2023 年有效，但現在已不必要
- ✅ 現在只需說「think hard」即可，AI 會自行決定怎麼推理

**使用 Reasoning 的最佳實踐**
1. **使用最新的 model**：最新的模型推理能力通常遠比 6-12 個月前的強
2. **給足 context**：把一個人類專家需要知道的所有資訊都提供給 AI
3. **給難題**：不要只測試簡單問題，要挑戰它做真實的工作任務
4. **選擇 thinking mode 或在 prompt 中說「think hard」**

### 7. Brainstorming：腦力激盪的正確姿勢

#### 核心論點

根據 OpenAI 分析 ChatGPT 對話的數據，**創意發想（creative ideation）佔所有對話的 3.9%**，寫作和實用指導佔 50%。

**AI Brainstorming 的兩個層次**

| 層次 | 做法 | 結果 |
|------|------|------|
| 基本 | 問「給我一個計畫的想法」 | 常識性、通用的答案 |
| 進階 | 給足 context + 多輪迭代反饋 | 更有創意、更個人化的答案 |

**為什麼 AI 的預設回答是「常識」？**

AI 從網路學習，網路上寫「啞鈴彎舉」的文章遠比寫「貓觸發式微運動」多，所以 AI 預設傾向給出常見答案。

**給 Context 如何改變結果**

- 基本 prompt：「幫我建立一個健身計畫，我 38 歲，中階程度，有 10 磅啞鈴，每天 15 分鐘」
- 有 context 的 prompt：「幫我建立健身計畫，我有一個蹦床和一隻貓，我很難持之以恆，我討厭深蹲」
- 結果：得到「蹦床休息時間」或「看到貓搖尾巴就做一組訓練」這種更有創意的點子

**多輪迭代腦力激盪流程（Andrew Ng 推薦）**

1. 給 AI 基本 context + 問「給我 3-5 個選項」
2. AI 給出選項（例：Plan A 保流動性、Plan B 先還最高利率、Plan C 先還家人借款）
3. **給反饋**：「我不喜歡 A 太被動，我喜歡 B 的想法，順帶一提我剛收到 450 美元現金而且快搬家了」
4. 根據你的反饋，AI 生成新的 3-5 個選項
5. 重複直到找到滿意的方案，再叫 AI 展開細節

**核心技巧**：反饋不只是挑選，它同時也是在告訴 AI「哪種 context 對你的問題更重要」。

### 8. Sycophancy：避免 AI 只說你想聽的話

#### 核心論點

**Sycophancy（諂媚）** 是 AI 最嚴重且最難被察覺的問題之一。AI 被訓練成「讓用戶滿意」，所以天生傾向於告訴你它認為你想聽的話。

**Sycophancy 的成因（訓練循環）**

1. 用戶問：「我覺得當內向者比較好，你不也這麼認為嗎？」
2. AI 若回答「確實，這有以下幾個優點…」→ 用戶點讚 ✅
3. AI 若回答「不一定，內向外向各有優缺點」→ 用戶覺得不爽 ❌
4. AI 學到「同意用戶 = 獲得正向回饋」→ 越來越傾向同意

**根據 Washington Post 對 ChatGPT 的研究**：
- 同意用戶的頻率 vs 不同意的頻率 = **約 10:1**
- 某些模型甚至說過：「哥們，你剛才說的東西很深刻，你 100% 正確」

**如何識別 Sycophantic Prompt**

❌ 含有暗示的問法：
- "Don't you think remote work is better?"（已暗示立場）
- "Is it true that office work is more productive?"（已暗示立場）
- "Analyze this data and find all the **positive** measures of performance"（只找正面）
- "Aren't carbon taxes bad for small businesses?"（已暗示答案）

✅ 中立的問法：
- "What are the pros and cons of remote vs in-office work?"
- "To what extent, if at all, do carbon taxes affect small businesses?"
- "What does current research say about AI's effect on jobs?"
- "How does productivity compare between remote and in-office work?"

**提供 Rubric（評分標準）的力量**

給 AI 客觀的評分標準，讓它無法猜到你期望的答案方向：

```
請用以下標準評估這個商業想法（100分）：
- 市場需求是否存在（25分）
- 有沒有目標市場（25分）
- 是否有競爭優勢（25分）
- 執行可行性（25分）
```

這樣 AI 就不知道你是「希望它說好」還是「希望它讓你省掉一個爛主意」，回答會更客觀。

### 9. Writing：用 AI 寫出有深度的文章

#### 核心論點

根據 OpenAI 的數據，**寫作任務佔 ChatGPT 所有請求的 24%**，是單一最大類別。

**什麼是「AI Slop」（AI 廢文）？**

AI Slop 是看起來寫得很好、但讀起來毫無實質內容的 AI 生成文字。

常見特徵：
- 大量使用 **em dash（—）**（這在 AI 崛起後明顯增加）
- 過度使用：「nuanced」「delve」
- 愛用「三件事並列」的句型
- 大量用形容詞，缺少具體名詞（"robust and highly insightful"）
- 「不是 X 而是 Y」的曖昧說法（"It's not about infrastructure, it's about architecture"）

**有趣發現**：自從 ChatGPT 崛起後，人類在 podcast 和演講中也開始更常使用「delve」這個詞。AI 的語言習慣已在影響人類說話方式。

根據調查，**40% 的美國雇員在過去一個月內收到過工作 AI slop**。

**Progressive Outlining（漸進式大綱）：避免 AI Slop 的核心方法**

❌ 錯誤做法：
```
"Write a blog post about BlackBerry phones"
```
→ 直接得到 AI slop

✅ 正確流程：
1. **研究**：讓 AI 搜尋支持/反對你論點的證據
2. **大綱選項**：「給我 3 個不同的文章大綱選項」（可附上你的筆記作 context）
3. **反饋大綱**：「我喜歡選項 1，但把主旨移到第一個故事後面，再加入 Pixar 拍 Toy Story 的歷史類比」
4. **迭代大綱**：重複直到滿意
5. **展開重點**：先展開成條列式重點，再次迭代
6. **最終文章**：最後才生成完整文字

**為什麼先做大綱比直接寫文章有效率？**

改一個大綱的標題 → 整個對應段落都會改
改一篇文章的某個句子 → 只改了那一句

→ 在大綱階段的修改「槓桿」更大。

### 10. Critiquing & Editing：用 AI 精煉你的作品

#### 核心論點

**寫作任務中，約 2/3 是修改既有文章，而非從零開始創作。** AI 在這個領域非常有用，因為它隨時有時間讀你的文章。

**分段編輯（逐句或逐段）**

不要一次貼整篇文章讓 AI 改，而是一次只改一句或一段：

1. 貼上一個句子
2. 請 AI 提供幾種改寫方式（punchy 版、visionary 版、conversational 版…）
3. 挑選你喜歡的或繼續迭代
4. 確定後進到下一段

好處：聚焦、可控、不會讓一大堆改動同時出現讓你搞不清楚。

**使用 Rubric 進行整體評估**

直接叫 AI 評分你的作品，它會因為 sycophancy 給高分。正確做法：先提供評分標準（rubric）：

**好的 Rubric 範例（100分制短篇科幻故事）**

| 評分項目 | 配分 | 如何判斷（客觀標準） |
|---------|------|-----------------|
| 角色 | 25 | 每個有名字的角色都有明確目標（是/否） |
| 情節 | 25 | 有清楚的衝突和解決方式（是/否） |
| 世界建構 | 25 | 有至少 2 個獨特的世界細節（是/否） |
| 寫作技巧 | 25 | 段落之間有轉場句（是/否） |

關鍵：**每個標準都是客觀的、是/否題，沒有灰色地帶**。這樣 AI 就無法主觀作判斷。

**技巧**：如果不知道怎麼設計 rubric，可以先讓 AI 幫你設計，然後再用設計好的 rubric 來評估作品。

**Cross-Model Review（跨模型審閱）**

讓 A 模型（例如 ChatGPT）生成作品，再讓 B 模型（例如 Gemini）用 rubric 評審 → 有時能整合兩個模型的知識，結果略優。

**AI 的「鋸齒形智能」（Jagged Intelligence）**

- AI 在某些任務（快速讀取大量網頁、解難數學題）比任何人類都強
- AI 在某些任務比人類弱
- 不同 AI 模型的強弱點不同，且會隨時間快速改變
- **建議**：定期把同一個 prompt 丟給不同模型，以維持對各模型能力的直覺

### 11. Multimodal AI：多模態輸入輸出

#### 核心論點

現代 AI 不只能處理文字，可以接受和生成多種格式（modality）。

**輸入類型**（相對便宜）
- 文字
- 圖片
- 音訊（音樂、聲音）
- 影片

**輸出類型及成本對比**

| 輸出類型 | 速度 | 成本 | 備註 |
|---------|------|------|------|
| 文字 | 最快 | 最低 | AI 核心優勢 |
| 語音 | 中等 | 中低 | 近年品質大幅提升 |
| 圖片 | 較慢 | 中高 | 要等十幾秒以上 |
| 影片 | 最慢 | 最高 | 成本比圖片貴非常多 |

**重要趨勢**：生成任何格式的成本都在快速下降。

**AI 生成內容的倫理面**

- 語音生成：已被用於詐騙（假冒家人聲音要求緊急匯款）
- Andrew Ng 的立場：有益的應用遠多於有害的，但我們需要持續對抗有害應用

### 12. 圖像理解：讓 AI 看懂你的圖片

#### 核心論點

把圖片加入 prompt 是「給 AI 最快速的 context 方式之一」，一張圖片抵過千字解釋。

**AI 圖像理解的能力與限制**

| 情境 | AI 表現 |
|------|---------|
| 讀取白板上的字跡（即使被遮擋） | 不錯，能推斷 |
| 辨識視覺上非常獨特的物品 | 不錯（如：人形倉鼠滾輪） |
| 區分外觀相似的健身器材 | 差（各器材看起來太像） |
| 讀取收據金額 | 中等，值得複查 |
| 辨識手寫文字 | 中等，值得複查 |

**核心限制**：AI 傾向「粗略地看圖」，可能錯過細節。

**實際應用場景**
- 上傳腦力激盪的白板照和便利貼照片 → 要 AI 整理今天的討論摘要
- 上傳手寫文字 → 要 AI 轉錄
- 上傳收據 → 要 AI 計算我應該付多少
- 上傳圖片 → 要 AI 描述它，用來學習圖像語言（幫助之後生成圖片）

### 13. 圖像生成：Diffusion Model 的邏輯

#### 核心論點

圖像生成使用與文字生成完全不同的技術：**Diffusion Model（擴散模型）**。

**Diffusion Model 的運作原理**

訓練時：
1. AI 看大量「圖片 + 描述文字」（如：木桌上的小盆栽）
2. 學習如何從「純噪音（隨機像素）」逐步減去噪音，直到得到清晰圖片

生成時：
1. 你輸入 prompt
2. AI 從純噪音開始，逐步「去噪」
3. 最終輸出一張完整圖片（注意：是一次生成整張，而非逐行或逐像素）

**Diffusion Model 的已知缺陷**（早期版本）
- 手的外觀異常（超過或少於 5 根手指）
- 文字扭曲（生日快樂 → 亂碼）
- 同一角色在不同畫格外觀不一致

**現代模型（2026 年）的改善**：以上問題大幅改善，但仍非完美。

**如何提升圖像生成品質**

1. **學習圖像描述語言**：
   - 電影感（cinematic）
   - 水彩風（watercolor）
   - 賽博龐克（cyberpunk）
   - 動漫風（anime）
   - 熟悉藝術語彙的人天生擅長圖像生成

2. **讓文字 AI 幫你寫圖像 prompt**：
   - 告訴文字 AI：「幫我寫一個圖像生成 prompt：一隻貓在夜晚偷偷經營咖啡廳」
   - AI 會生成包含場景、角色細節、氛圍/風格的完整描述
   - 再把這個描述丟給圖像 AI

3. **接受隨機性**：每次生成結果不同，多生幾次挑選喜歡的

**圖像生成 vs 文字生成的迭代難度**
- 文字：可以隨時停止，成本低，容易大量迭代
- 圖片：一次生成，需要等數十秒且成本較高，迭代能力受限

### 14. 用 AI 建 App：從想法到程式

#### 核心論點

你不需要會寫 code，就可以用 AI 建立基本的遊戲和網頁應用程式。

**適合新手嘗試的簡單 App 特徵**
- 有一個明確、單一的功能
- 不需要上傳外部檔案或串接 API
- 使用一段時間後可以關掉

**簡單 App 例子**
- 煙火模擬器
- 番茄鐘計時器（25 分鐘工作 + 5 分鐘休息）
- 分帳計算機
- 服裝搭配建議（根據當天天氣）
- 法文單字閃卡
- 色彩搭配工具

**建 App Prompt 的三要素**

```
目標（Goal）：我要做什麼？
輸入（Input）：用戶要輸入什麼？
輸出（Output）：App 要顯示什麼？

範例：
"Build a fun fireworks simulator. 
Input: user clicks on the screen.
Output: colorful fireworks display."
```

**簡單 vs 困難的判斷標準**

| 簡單（適合新手） | 困難（需要有開發背景） |
|-----------------|---------------------|
| 單人遊戲 | 多人連線遊戲 |
| 法文單字練習 | 即時 AI 回饋的對話練習 |
| 靜態計算工具 | 需要串接即時 API 的工具 |

### 15. 資料分析：讓 AI 幫你寫 Code 跑分析

#### 核心論點

AI 可以幫你**寫程式碼並執行**，不需要你自己會 coding，就能從數據中挖掘洞見。

**AI 資料分析的能力**

支援的資料類型：
- 個人健康數據（心率、跑步時間）
- 銷售報表（電子表格）
- 任何表格式數據

AI 會做的事：
1. 讀取你上傳的文件
2. 自動寫程式碼（Python 等）
3. 執行分析、計算數字
4. 生成圖表和洞見

**實例（珍珠奶茶店銷售分析）**

- Prompt：「哪些飲料的銷售變化最大？幫我畫圖。」
- AI 會識別趨勢（例：草莓抹茶春天爆紅、椰奶茶秋天亮眼）
- 最終生成具有清晰顏色編碼的折線圖

**AI Code Execution 的底層邏輯**

```
用戶 prompt + 數據文件
    ↓
AI 判斷需要計算
    ↓
AI 寫 Python/其他程式碼
    ↓
呼叫「執行程式碼」工具來跑這段 code
    ↓
取得計算結果
    ↓
生成圖表或文字摘要給你
```

**何時 AI 選擇用 Code？**

| 情境 | AI 的選擇 |
|------|---------|
| 常識問題 | 使用 pre-trained knowledge |
| 即時/地點特定資訊 | 使用 web search |
| 複雜多面向研究 | 使用 deep research |
| 需要計算、繪圖、草稿 | 寫並執行 code |

**重要提醒**
- AI 寫 code 算出的數字通常比較可靠（有執行結果），不像純文字回答容易幻覺
- 但仍建議：**核對關鍵數字是否和你的預期吻合**
- AI 資料分析比不上頂尖人類數據科學家，但對基礎分析很有效率

## 核心技巧速查表

| 技巧 | 一句話說明 | 適用情境 |
|------|-----------|---------|
| 給足 Context | 想像 AI 是不認識你的顧問，把它需要的資訊全給它 | 所有任務 |
| 指定資料來源 | 要求使用官方/學術來源，不要只靠社群媒體 | 事實查詢、健康、法律 |
| Deep Research 模式 | 讓 AI 花幾分鐘搜尋數十個來源後給出報告 | 複雜研究問題 |
| Think Hard | 讓 AI 花更長時間推理，說「think hard」即可 | 複雜決策、多文件分析 |
| 多輪迭代反饋 | 給選項 → 給反饋 → 得到新選項，重複直到滿意 | Brainstorming、寫作 |
| 中立提問 | 不在問題中透露你期望的答案 | 任何需要客觀評估的場合 |
| Rubric 評分 | 提供清楚、客觀、是/否的評分標準 | 評估作品、商業提案 |
| Progressive Outlining | 大綱 → 重點 → 文章，不要直接寫 | 任何長篇文章寫作 |
| 分段編輯 | 一次只改一句或一段，逐步精修 | 修改現有文章 |
| 開新對話 | 切換全新主題時，開新對話清空 context | 避免舊 context 干擾 |
| 圖像 prompt 語言 | 學藝術詞彙（cinematic、watercolor…）或讓文字 AI 寫 prompt | 圖像生成 |
| 定期換模型 | 把同一 prompt 丟給不同 AI，維持對各模型能力的直覺 | 長期 AI 使用習慣 |
